How Adaptive Streaming Works

┌──────────┐ transcode ┌────────────────────────────────┐ │ OGG File │ ──────────────▶ │ Multiple Bitrate Variants │ └──────────┘ (ffmpeg) │ 64k │ 128k │ 256k │ └───┬───┴────┬───┴────┬──────────┘ │ │ │ ┌───▼────────▼────────▼──────────┐ │ Segmenter (4-second chunks) │ └───┬────────┬────────┬──────────┘ │ │ │ ┌────────────▼────────▼────────▼──────────┐ │ HTTP Server (Express) │ │ Serves manifests + segments on demand │ └────────────────┬────────────────────────┘ │ ┌────────────────▼────────────────────────┐ │ Browser Player (hls.js/dash.js) │ │ • Fetches manifest │ │ • Monitors bandwidth + buffer │ │ • Switches quality automatically │ └─────────────────────────────────────────┘
  • The source audio is encoded at 3 quality tiers: 64 kbps (low), 128 kbps (mid), 256 kbps (high).
  • Each tier is split into 4-second segments so the player can switch quality at segment boundaries.
  • HLS uses .m3u8 playlists and .ts segments. A master playlist lists all variants.
  • DASH uses an .mpd XML manifest and .m4s segments with adaptation sets.
  • The player's ABR algorithm measures download speed and buffer fullness, then picks the highest quality that won't cause buffering.
  • Quality switches happen seamlessly at segment boundaries — no interruption to playback.
